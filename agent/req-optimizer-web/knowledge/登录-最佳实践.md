# 登录与账号体系最佳实践

## 密码与传输
- 密码必须在客户端用 HTTPS 传输，服务端用 bcrypt（cost ≥ 10）或 argon2id 存储
- 严禁使用 MD5 / SHA1 / 明文存储
- 密码复杂度：长度 ≥ 8，至少包含字母 + 数字两类，禁止 top1000 弱密码

## 防暴力破解
- 同一账号连续失败 5 次 → 锁定 15 分钟
- 同一 IP 1 分钟内 ≥ 20 次失败 → 触发图形/滑块验证码
- 失败提示统一为"账号或密码错误"，不可暴露"该账号不存在"

## 会话与"记住我"
- Access Token 有效期建议 ≤ 2h，Refresh Token ≤ 30d
- "记住我"= 持久化 Refresh Token（HttpOnly + Secure + SameSite=Lax）
- Token 必须绑定设备指纹或 user-agent，登出时双端失效

## 密码重置
- 重置链接 / 验证码有效期 ≤ 24h，单次有效，使用后立即失效
- 重置流程必须二次确认旧会话登出
- 通过邮箱重置时，发送链接前不应区分"邮箱是否注册"，避免账号枚举

## 多端策略
- 同账号最多保留 N 个活跃会话（建议 5），超过则踢掉最早的
- 提供"查看登录设备"与"远程登出"
