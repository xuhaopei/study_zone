# 用户登录功能 · 优化需求文档

> 文档版本：v1.0 · 生成时间：2026-06-22

---

## 一、需求背景

当前产品需要为用户提供统一的身份认证入口。原始需求描述过于笼统（"登录要快速""密码要安全一点"），缺乏可量化指标与安全规范约束，无法直接指导研发与测试。本文档在原始需求基础上，结合内部安全规范、合规要求与性能基线，对功能边界、安全约束、验收标准进行全面补全。

**核心目标：**
1. 提供账号密码登录与手机号验证码登录两种方式；
2. 满足安全合规要求（密码存储、防暴力破解、会话管理、PIPL 合规）；
3. 登录页面与接口达到既定性能基线；
4. 登录成功后跳转首页并展示欢迎提示。

---

## 二、目标用户与使用场景

| 用户群体 | 典型场景 | 设备 / 网络 |
|---|---|---|
| 普通注册用户 | 账号 + 密码日常登录 | PC 浏览器、移动端 4G/Wi-Fi |
| 手机号用户 | 忘记密码时用手机号验证码快捷登录 | 移动端为主 |
| 高频用户 | 勾选"记住我"免密复访 | 固定设备 |
| 安全审计人员 | 查看登录日志、远程登出异常设备 | 后台管理端 |

**不在本期范围内：** 第三方 OAuth 登录（微信/Apple）、生物识别登录、企业 SSO。

## 三、功能需求（FR）

### FR-01 账号密码登录

| 项目 | 内容 |
|---|---|
| **描述** | 用户在登录页输入账号（用户名或邮箱）与密码，通过身份验证后进入系统。 |
| **输入** | `account`（字符串，用户名或邮箱）、`password`（字符串，明文，HTTPS 传输） |
| **输出** | 登录成功：返回 Access Token + Refresh Token，前端跳转首页并弹出欢迎提示；登录失败：返回统一错误提示。 |
| **约束** | ① 密码在服务端以 **bcrypt（cost ≥ 10）** 或 argon2id 存储，严禁 MD5/SHA1/明文。② 全程强制 **HTTPS** 传输。③ 密码复杂度要求：长度 ≥ 8，至少包含字母 + 数字两类，禁止 top1000 弱密码（注册/改密时校验）。 |
| **异常分支** | ① 账号不存在或密码错误：统一提示"账号或密码错误"，**不可区分**两种情况（防账号枚举）。② 账号被锁定：提示"账号已锁定，请 15 分钟后重试或联系客服"。③ 网络超时：提示"网络异常，请稍后重试"，不清空已输入内容。 |

---

### FR-02 防暴力破解

| 项目 | 内容 |
|---|---|
| **描述** | 对高频失败登录行为进行自动拦截，保护账号安全。 |
| **输入** | 登录请求中的账号标识 + 客户端 IP |
| **输出** | 触发阈值后返回锁定状态或验证码挑战，并记录安全日志。 |
| **约束** | ① **同一账号**连续失败 **5 次** → 账号锁定 **15 分钟**，锁定期间拒绝所有登录请求。② **同一 IP** 1 分钟内失败 **≥ 20 次** → 强制触发图形/滑块验证码，验证通过后方可继续。③ 锁定计数在锁定期满后自动重置。 |
| **异常分支** | ① 验证码加载失败：提供"刷新验证码"入口，不阻断登录流程。② 用户申诉解锁：通过手机号验证码或人工客服渠道提前解锁。 |

---

### FR-03 手机号验证码登录

| 项目 | 内容 |
|---|---|
| **描述** | 用户输入手机号，接收短信验证码后完成登录（同时作为忘记密码的快捷通道）。 |
| **输入** | `phone`（11 位大陆手机号）、`sms_code`（6 位数字验证码） |
| **输出** | 验证通过：同 FR-01 返回 Token 并跳转首页；验证失败：提示错误原因。 |
| **约束** | ① 手机号属于**敏感个人信息**，需加密存储，收集前须取得用户**单独同意**，并在隐私政策中明示采集字段、用途、存储期限。② 验证码有效期 ≤ **10 分钟**，单次有效，使用后立即失效。③ 同一手机号发送频率限制：60 秒内不可重复发送，每日上限 10 条。④ 验证码不得在日志中明文记录。 |
| **异常分支** | ① 手机号未注册：提示"该手机号未绑定账号，请先注册"。② 验证码过期：提示"验证码已失效，请重新获取"。③ 短信通道故障：提示"短信发送失败，请稍后重试"，并上报告警。 |

---

### FR-04 会话管理与"记住我"

| 项目 | 内容 |
|---|---|
| **描述** | 管理用户登录态的生命周期，支持"记住我"持久化登录及多端会话控制。 |
| **输入** | 登录成功事件、用户勾选"记住我"选项、登出请求 |
| **输出** | 颁发 Token、维护会话列表、登出时双端失效。 |
| **约束** | ① **Access Token** 有效期 ≤ **2 小时**；**Refresh Token** 有效期 ≤ **30 天**。② "记住我"= 持久化 Refresh Token，Cookie 属性必须设置 `HttpOnly + Secure + SameSite=Lax`。③ Token 必须绑定设备指纹或 User-Agent，防止 Token 被盗用。④ 同一账号最多保留 **5 个**活跃会话，超出后自动踢掉最早的会话。⑤ 登出时 Access Token 与 Refresh Token **双端同时失效**。 |
| **异常分支** | ① Token 过期：静默刷新失败后跳转登录页，保留当前页面 URL 供登录后回跳。② 设备指纹不匹配：强制重新登录并发送安全通知。 |

---

### FR-05 登录成功后首页欢迎提示

| 项目 | 内容 |
|---|---|
| **描述** | 用户登录成功跳转首页后，展示个性化欢迎提示信息。 |
| **输入** | 用户昵称/姓名、登录时间、上次登录 IP（脱敏） |
| **输出** | 首页顶部或浮层展示"欢迎回来，{昵称}！上次登录时间：{时间}"，3~5 秒后自动消失，支持手动关闭。 |
| **约束** | ① 提示信息不得展示完整手机号或邮箱（需脱敏，如 `138****8888`）。② 首次登录用户展示"欢迎加入！"引导语，不展示上次登录信息。 |
| **异常分支** | ① 用户信息接口超时：跳过提示，不阻塞首页主体内容加载。 |

---

### FR-06 登录日志与审计

| 项目 | 内容 |
|---|---|
| **描述** | 记录所有登录行为（成功与失败），供安全审计与用户自查使用。 |
| **输入** | 每次登录请求（账号、IP、设备、时间、结果） |
| **输出** | 写入审计日志；用户可在"安全中心"查看登录记录；提供"远程登出"操作。 |
| **约束** | ① 日志保留期 ≥ **180 天**。② 日志中**严禁**记录密码、完整手机号、完整身份证号。③ 提供"查看登录设备"与"远程登出"功能入口。 |
| **异常分支** | ① 日志写入失败：不影响登录主流程，异步重试并告警。 |

## 四、非功能需求（NFR）

### NFR-01 性能

| 指标 | 目标值 | 说明 |
|---|---|---|
| 登录页 LCP | ≤ 2.5s | Web Vitals 移动端优先，4G 网络 |
| 登录页首屏 TTI | ≤ 3s | 4G 网络条件下 |
| 登录页 CLS | ≤ 0.1 | 防止布局抖动影响输入体验 |
| 登录接口 P95 响应时间 | ≤ 500ms | 后端接口达标线 |
| 登录接口 P99 响应时间 | ≤ 1s | 后端接口上限 |
| 接口 5xx 错误率 | ≤ 0.1% | 可用性基线 |
| INP（交互响应） | ≤ 200ms | 按钮点击到视觉反馈 |

### NFR-02 安全

- 全站强制 HTTPS，禁止 HTTP 访问登录接口。
- 密码存储：bcrypt（cost ≥ 10）或 argon2id，严禁 MD5/SHA1/明文。
- 防暴力破解：账号锁定 + IP 频率限制（见 FR-02）。
- XSS/CSRF 防护：登录表单启用 CSRF Token，输出内容做 HTML 转义。

### NFR-03 合规与隐私

- 手机号等敏感个人信息收集前须明示用途并取得单独同意（PIPL 要求）。
- 用户享有查询、更正、删除、导出 4 项基本权利，需提供对应入口。
- 仅收集登录业务所需最小字段，禁止"以备将来用"的额外采集。
- 如使用境外云服务存储用户数据，需完成数据出境安全评估。

### NFR-04 兼容性

| 维度 | 要求 |
|---|---|
| 桌面浏览器 | Chrome / Edge / Safari 最近 2 个大版本，Firefox ESR |
| 移动端系统 | iOS ≥ 14，Android ≥ 8 |
| 屏幕宽度 | ≥ 360px，需覆盖 360 / 375 / 414 / 768 / 1280 五档测试 |

### NFR-05 可用性

- 登录服务可用性 ≥ 99.9%（月度）。
- 短信通道具备主备切换能力，单通道故障不影响整体登录。

## 五、验收标准

### AC-01 账号密码登录成功

```
Given 已注册用户，账号密码正确
When  在登录页输入账号和密码，点击"登录"按钮
Then  ① 2s 内跳转至首页
      ② 首页顶部展示"欢迎回来，{昵称}！"提示，3~5s 后自动消失
      ③ 浏览器 Cookie 中存在 HttpOnly+Secure+SameSite=Lax 的 Refresh Token
```

### AC-02 密码错误提示（防账号枚举）

```
Given 已注册账号，输入错误密码（累计 < 5 次）
When  点击"登录"
Then  ① 页面提示"账号或密码错误"，不区分账号是否存在
      ② 不跳转，密码输入框内容清空，账号输入框保留
```

### AC-03 账号锁定

```
Given 同一账号连续登录失败 5 次
When  第 5 次失败后
Then  ① 账号被锁定 15 分钟
      ② 页面提示"账号已锁定，请 15 分钟后重试"
      ③ 锁定期间任何登录尝试均返回锁定提示，不消耗失败计数
```

### AC-04 IP 频率触发验证码

```
Given 同一 IP 在 1 分钟内登录失败 ≥ 20 次
When  第 20 次失败后
Then  后续请求强制展示图形/滑块验证码，验证通过后方可提交登录
```

### AC-05 手机号验证码登录

```
Given 已绑定手机号的用户
When  输入手机号，点击"获取验证码"，收到短信后输入 6 位验证码，点击"登录"
Then  ① 验证码 10 分钟内有效，使用后立即失效
      ② 验证通过后跳转首页，流程与 AC-01 一致
      ③ 60 秒内重复点击"获取验证码"按钮置灰，倒计时展示
```

### AC-06 会话过期回跳

```
Given 用户 Access Token 已过期且 Refresh Token 也已失效
When  用户访问任意需登录页面
Then  ① 自动跳转登录页
      ② URL 中携带 redirect 参数记录原页面地址
      ③ 登录成功后自动跳回原页面
```

### AC-07 登录页性能

```
Given 移动端 4G 网络，Chrome 最新版
When  首次访问登录页（无缓存）
Then  ① LCP ≤ 2.5s
      ② TTI ≤ 3s
      ③ CLS ≤ 0.1
      ④ 登录接口 P95 响应时间 ≤ 500ms
```

### AC-08 日志合规

```
Given 用户完成一次登录（成功或失败）
When  查询审计日志
Then  ① 日志包含：账号（脱敏）、IP、设备、时间、结果
      ② 日志中不含明文密码、完整手机号
      ③ 日志可查询，保留期 ≥ 180 天
```
